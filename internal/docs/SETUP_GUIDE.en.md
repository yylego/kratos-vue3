## Overview

This guide walks through the complete setup: generating TypeScript gRPC clients from Kratos proto files, converting them to HTTP clients, and integrating into a Vue 3 frontend.

```
.proto files  →  protoc + @protobuf-ts/plugin  →  gRPC TS Client  →  vue3kratos CLI  →  HTTP TS Client
```

---

## Prerequisites

### Install @protobuf-ts/plugin

Install the protoc TypeScript plugin at system scope:

```bash
npm install -g @protobuf-ts/plugin
```

Confirm the executable is available:

```
$ which protoc-gen-ts
/Users/admin/.nvm/versions/node/v18.17.0/bin/protoc-gen-ts
```

> **Warning**: Do NOT use the [protoc-gen-ts](https://www.npmjs.com/package/protoc-gen-ts) package (same executable name, different package). This guide requires [`@protobuf-ts/plugin`](https://www.npmjs.com/package/@protobuf-ts/plugin) — using a different package will cause issues in subsequent steps.

### Install vue3kratos CLI

```bash
go install github.com/yylego/kratos-vue3/cmd/vue3kratos@latest
```

## Step 1: Generate TypeScript gRPC Client

Use `protoc` with `protoc-gen-ts` to generate TypeScript client code from `.proto` files.

### Makefile Configuration

Add the following to the Kratos project Makefile:

```makefile
THIRD_PARTY_GOOGLE_API_PROTO_FILES=$(shell find third_party/google/api -name *.proto)

web_api_grpc_ts:
	mkdir -p ./bin/web_api_grpc_ts.out
	PROTOC_GEN_TS=$$(which protoc-gen-ts) && \
	protoc \
	--plugin=protoc-gen-ts=$$PROTOC_GEN_TS \
	--ts_out=./bin/web_api_grpc_ts.out \
	--proto_path=./api \
	--proto_path=./third_party \
	$(API_PROTO_FILES)

	PROTOC_GEN_TS=$$(which protoc-gen-ts) && \
	protoc \
	--plugin=protoc-gen-ts=$$PROTOC_GEN_TS \
	--ts_out=./bin/web_api_grpc_ts.out \
	--proto_path=./third_party \
	$(THIRD_PARTY_GOOGLE_API_PROTO_FILES)
```

The template above uses `$(which protoc-gen-ts)` to auto-detect the plugin path. If needed, replace it with an absolute path matching the machine:

| Platform | Example Path |
|----------|-------------|
| macOS (Node v18) | `/Users/admin/.nvm/versions/node/v18.17.0/bin/protoc-gen-ts` |
| Linux (Node v20) | `/home/yangyile/.nvm/versions/node/v20.14.0/bin/protoc-gen-ts` |

When upgrading Node.js, update this path to the new version.

> Complete Makefile examples: [demo1](https://github.com/yylego/kratos-vue3-demos/blob/main/demo1kratos/Makefile) | [demo2](https://github.com/yylego/kratos-vue3-demos/blob/main/demo2kratos/Makefile)

### Run Code Generation

```bash
# 1. Create a Kratos project (if starting fresh)
kratos new helloworld

# 2. Execute generation
make web_api_grpc_ts
```

Confirm the output in `bin/web_api_grpc_ts.out`:

```
$ ls bin/web_api_grpc_ts.out/helloworld/v1/
error_reason.ts    greeter.client.ts    greeter.ts
```

The generated `*.client.ts` files contain gRPC client code with `// @generated by protobuf-ts` at the top.

## About the Conversion

The generated code uses the gRPC protocol. Browsers cannot make native gRPC calls, so the code must be converted to HTTP.

One option is a gRPC-web proxy, but that requires additional infrastructure. This project takes a different approach: the `vue3kratos` CLI rewrites the client code to convert gRPC invocations into `axios`-based HTTP requests. Benefits:

- Same function invocation pattern as the original gRPC client
- Complete TypeScript type checking on parameters and responses
- No proxy or bridge infrastructure needed

## Step 2: Convert gRPC Client → HTTP Client

### Method A: Go Side (Recommended)

Run the conversion as part of the backend build pipeline. This automates the process on each proto change.

Convert a single file:

```bash
vue3kratos gen-grpc-via-http-in-path --grpc-ts-path=/path/to/generated.client.ts
```

> **Warning**: This command modifies the target file in-place. Ensure the path is correct.

### Method B: Vue Side

Run the conversion in the frontend project. See the demo Makefiles:

- [demo1 Makefile](https://github.com/yylego/kratos-vue3-demos/blob/main/demo1kratos/Makefile)
- [demo2 Makefile](https://github.com/yylego/kratos-vue3-demos/blob/main/demo2kratos/Makefile)

---

## Step 3: Install Runtime Package

Both conversion methods above produce code that depends on the [`@yylego/grpc-to-http`](https://www.npmjs.com/package/@yylego/grpc-to-http) runtime package. Install it in the frontend project:

```bash
npm install @yylego/grpc-to-http
```

This package handles the actual gRPC-to-HTTP request translation at runtime.

- NPM: [@yylego/grpc-to-http](https://www.npmjs.com/package/@yylego/grpc-to-http)
- GitHub: [yylego/grpc-to-http](https://github.com/yylego/grpc-to-http)

## Recommended Workflow

```
1. Update .proto files
2. make web_api_grpc_ts          → Generate TypeScript gRPC client
3. vue3kratos gen-grpc-via-http  → Convert to HTTP client
4. Copy output to frontend       → Use in Vue 3 project
```

This keeps frontend code in sync with proto changes via automation.
