# vue3kratos 配置指南

Vue 3 前端与 Kratos 后端集成工具包。

---

[ENGLISH README](SETUP_GUIDE.en.md)

---

## 概述

本指南完整介绍：从 Kratos proto 文件生成 TypeScript gRPC 客户端，转换为 HTTP 客户端，并集成到 Vue 3 前端项目。

```
.proto 文件  →  protoc + @protobuf-ts/plugin  →  gRPC TS 客户端  →  vue3kratos CLI  →  HTTP TS 客户端
```

---

## 环境准备

### 安装 @protobuf-ts/plugin

在系统范围安装 protoc TypeScript 插件：

```bash
npm install -g @protobuf-ts/plugin
```

确认可执行文件可用：

```
$ which protoc-gen-ts
/Users/admin/.nvm/versions/node/v18.17.0/bin/protoc-gen-ts
```

> **注意**：请不要使用 [protoc-gen-ts](https://www.npmjs.com/package/protoc-gen-ts) 这个同名包。本文档基于 [`@protobuf-ts/plugin`](https://www.npmjs.com/package/@protobuf-ts/plugin)，使用不同的包会导致后续步骤出错。

### 安装 vue3kratos CLI

```bash
go install github.com/yylego/kratos-vue3/cmd/vue3kratos@latest
```

## 第一步：生成 TypeScript gRPC 客户端

使用 `protoc` 配合 `protoc-gen-ts` 从 `.proto` 文件生成 TypeScript 客户端代码。

### Makefile 配置

在 Kratos 项目的 Makefile 中添加：

```makefile
THIRD_PARTY_GOOGLE_API_PROTO_FILES=$(shell find third_party/google/api -name *.proto)

web_api_grpc_ts:
	mkdir -p ./bin/web_api_grpc_ts.out
	PROTOC_GEN_TS=$$(which protoc-gen-ts) && \
	protoc \
	--plugin=protoc-gen-ts=$$PROTOC_GEN_TS \
	--ts_out=./bin/web_api_grpc_ts.out \
	--proto_path=./api \
	--proto_path=./third_party \
	$(API_PROTO_FILES)

	PROTOC_GEN_TS=$$(which protoc-gen-ts) && \
	protoc \
	--plugin=protoc-gen-ts=$$PROTOC_GEN_TS \
	--ts_out=./bin/web_api_grpc_ts.out \
	--proto_path=./third_party \
	$(THIRD_PARTY_GOOGLE_API_PROTO_FILES)
```

上述模板使用 `$(which protoc-gen-ts)` 自动检测插件路径。如需指定绝对路径，请根据机器环境替换：

| 平台 | 路径示例 |
|------|---------|
| macOS (Node v18) | `/Users/admin/.nvm/versions/node/v18.17.0/bin/protoc-gen-ts` |
| Linux (Node v20) | `/home/yangyile/.nvm/versions/node/v20.14.0/bin/protoc-gen-ts` |

升级 Node.js 版本后，需要同步更新此路径。

> 完整 Makefile 示例：[demo1](https://github.com/yylego/kratos-vue3-demos/blob/main/demo1kratos/Makefile) | [demo2](https://github.com/yylego/kratos-vue3-demos/blob/main/demo2kratos/Makefile)

### 执行代码生成

```bash
# 1. 创建 Kratos 项目（如果是新项目）
kratos new helloworld

# 2. 执行生成命令
make web_api_grpc_ts
```

确认 `bin/web_api_grpc_ts.out` 中的输出：

```
$ ls bin/web_api_grpc_ts.out/helloworld/v1/
error_reason.ts    greeter.client.ts    greeter.ts
```

生成的 `*.client.ts` 文件包含 gRPC 客户端代码，文件头部标注 `// @generated by protobuf-ts`。

## 关于转换

生成的代码使用 gRPC 协议。浏览器无法直接发起 gRPC 调用，因此需要转换为 HTTP。

一种方案是 gRPC-web 代理，但需要额外的基础设施配置。本项目采用另一种方案：`vue3kratos` CLI 重写客户端代码，将 gRPC 调用转换为基于 `axios` 的 HTTP 请求。优势：

- 保持与原始 gRPC 客户端相同的函数调用模式
- 参数和返回值均带有 TypeScript 类型检查
- 无需额外的代理或桥接基础设施

## 第二步：gRPC 客户端 → HTTP 客户端

### 方式 A：Go 侧转换（推荐）

在后端构建流程中完成转换，每次 proto 变更时自动执行。

转换单个文件：

```bash
vue3kratos gen-grpc-via-http-in-path --grpc-ts-path=/path/to/generated.client.ts
```

> **注意**：该命令会原地修改目标文件，请确保路径正确。

### 方式 B：Vue 侧转换

在前端项目中完成转换。参考演示项目的 Makefile：

- [demo1 Makefile](https://github.com/yylego/kratos-vue3-demos/blob/main/demo1kratos/Makefile)
- [demo2 Makefile](https://github.com/yylego/kratos-vue3-demos/blob/main/demo2kratos/Makefile)

## 第三步：安装运行时依赖

以上两种转换方式生成的代码都依赖 [`@yylego/grpc-to-http`](https://www.npmjs.com/package/@yylego/grpc-to-http) 运行时包。在前端项目中安装：

```bash
npm install @yylego/grpc-to-http
```

该包负责在运行时将 gRPC 调用转换为 HTTP 请求。

- NPM：[@yylego/grpc-to-http](https://www.npmjs.com/package/@yylego/grpc-to-http)
- GitHub：[yylego/grpc-to-http](https://github.com/yylego/grpc-to-http)

---

## 推荐工作流

```
1. 修改 .proto 文件
2. make web_api_grpc_ts          → 生成 TypeScript gRPC 客户端
3. vue3kratos gen-grpc-via-http  → 转换为 HTTP 客户端
4. 将输出复制到前端项目          → 在 Vue 3 中使用
```

这样每次 proto 变更后，前端代码都能自动同步更新，保持一致。
